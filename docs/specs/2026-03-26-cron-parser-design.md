# Cron 表达式解析工具 — 设计文档

**日期:** 2026-03-26
**状态:** 已批准
**工具 ID:** `cron-parser`

---

## 1. 需求概述

在 dev-tools 工具箱中新增一个 Cron 表达式解析工具，支持：

- 多种 Cron 格式：Linux 5/6 位、Quartz/Spring 6/7 位
- 字段可视化 + 人类可读描述
- 接下来 5 次执行时间预测
- 交互式字段选择器（双向编辑）
- 常见预设快速选择
- 数据全程本地处理，不上传服务器

---

## 2. 功能规格

### 2.1 操作模式

| 操作 | 触发方式 | 输出 |
|------|---------|------|
| 输入 cron | 手动输入或 UI 选择 | 实时解析 + 字段高亮 + 描述 |
| 解析 | 输入变化时自动触发 | 字段卡片 + 人类可读描述 |
| 清除 | 点击清除按钮 | 重置所有输入和结果 |

### 2.2 支持的 cron 格式

| 格式 | 字段数 | 字段顺序 | 特殊字符 | 示例 |
|------|--------|---------|---------|------|
| Linux 5位 | 5 | 分 时 日 月 周 | `* - , /` | `*/5 * * * *` |
| Linux 6位 | 6 | 秒 分 时 日 月 周 | `* - , /` | `0 */5 * * * *` |
| Quartz 6位 | 6 | 秒 分 时 日 月 周 | `* - , / ? L W #` | `0 0/5 * ? * *` |
| Quartz 7位 | 7 | 秒 分 时 日 月 周 年 | `* - , / ? L W #` | `0 0/5 * ? * * *` |

**格式选择**：用户通过下拉框选择格式，字段数量动态调整。

**Quartz 特殊字符说明**：

| 字符 | 用途 | 适用字段 | 说明 |
|------|------|---------|------|
| `?` | 不指定 | 日、周 | 日和周字段不能同时为 `*` |
| `L` | 最后 | 日、周 | `L` = 最后一天，`6L` = 最后一个周五 |
| `W` | 工作日 | 日 | `15W` = 15号最近的工作日 |
| `#` | 第N个 | 周 | `6#3` = 第3个周五 |

### 2.3 字段支持的语法

**标准语法（所有格式）**：

| 语法 | 说明 | 示例 |
|------|------|------|
| `*` | 任意值 | 每分钟 |
| `*/N` | 每 N 个单位 | `*/5` = 每 5 分钟 |
| `N` | 具体值 | `30` = 第 30 分钟 |
| `N-M` | 范围 | `1-5` = 1 到 5 |
| `N,M,O` | 列表 | `1,3,5` = 第 1、3、5 分钟 |
| `N-M/O` | 范围+步长 | `1-30/5` = 1-30 每 5 分钟 |

**Quartz 特殊语法（仅 Quartz 格式）**：

| 语法 | 说明 | 示例 |
|------|------|------|
| `?` | 不指定（日或周） | `0 0 12 ? * *` = 每天中午，周不指定 |
| `L` | 最后 | `0 0 0 L * ?` = 每月最后一天 |
| `dL` | 最后一个周d | `0 0 0 ? * 6L` = 最后一个周五 |
| `dW` | 最近工作日 | `0 0 0 15W * ?` = 15号最近工作日 |
| `d#n` | 第n个周d | `0 0 0 ? * 6#3` = 第3个周五 |

### 2.4 反向生成 Cron（双向编辑）

| 操作 | 说明 |
|------|------|
| UI 选择字段 | 通过下拉框/输入框选择每个字段值 |
| 自动生成 cron | UI 变化时实时生成 cron 表达式 |
| 同步 | cron 输入框 ↔ UI 选择器双向同步 |

**交互流程**：
1. 用户通过 UI 选择：分钟 = `*/5`，其他 = `*`
2. 自动生成：`*/5 * * * *`
3. 用户手动修改 cron 为：`0 */5 * * *`
4. UI 自动更新：秒 = `0`，分钟 = `*/5`，其他 = `*`

### 2.5 常见预设

| 预设 | Cron | 描述 |
|------|------|------|
| 每分钟 | `* * * * *` | 每分钟执行 |
| 每小时 | `0 * * * *` | 每小时整点 |
| 每天午夜 | `0 0 * * *` | 每天 00:00 |
| 每周一 | `0 0 * * 1` | 每周一 00:00 |
| 工作日 | `0 9 * * 1-5` | 工作日上午 9 点 |
| 每 5 分钟 | `*/5 * * * *` | 每 5 分钟 |

### 2.6 边界条件

| 场景 | 处理方式 |
|------|---------|
| 空输入 | 不解析，显示提示 |
| 非法格式 | 红色边框 + 错误提示 |
| 字段数不是 5 或 6 | 提示"请输入 5 位或 6 位 cron 表达式" |
| 字段值超出范围 | 高亮错误字段 + 提示有效范围 |

## 4. 代码架构

### 4.1 文件结构

```
src/tools/cron-parser/
├── CronParser.vue        # 页面组件
├── cron.ts               # 纯工具函数（解析、生成、时间预测）
└── presets.ts            # 常见预设定义
```

### 4.2 工具函数接口（`cron.ts`）

```ts
/**
 * Cron 字段解析结果
 */
export interface CronField {
  value: string           // 原始值，如 "*/5"
  label: string           // 字段名称，如 "分"
  description: string     // 描述，如 "每 5 分钟"
  valid: boolean          // 是否有效
  error?: string          // 错误信息
}

/**
 * Cron 解析结果
 */
export interface CronParseResult {
  success: boolean
  fields?: CronField[]    // 5 或 6 个字段
  description?: string    // 人类可读描述
  nextRuns?: Date[]       // 接下来 5 次执行时间
  error?: string
}

/**
 * 解析 cron 表达式
 * 自动识别 5 位或 6 位格式
 * @param expression - cron 表达式
 * @param baseTime - 基准时间（默认当前时间）
 */
export function parseCron(
  expression: string,
  baseTime?: Date
): CronParseResult

/**
 * 从字段值生成 cron 表达式
 * @param fields - 字段数组 [秒(可选), 分, 时, 日, 月, 周]
 */
export function generateCron(fields: string[]): string

/**
 * 验证单个字段值是否有效
 * @param value - 字段值
 * @param fieldType - 字段类型
 */
export function validateField(
  value: string,
  fieldType: 'second' | 'minute' | 'hour' | 'day' | 'month' | 'weekday'
): { valid: boolean; error?: string }

/**
 * 计算接下来 N 次执行时间
 * @param expression - cron 表达式
 * @param baseTime - 基准时间
 * @param count - 次数（默认 5）
 */
export function getNextRuns(
  expression: string,
  baseTime?: Date,
  count?: number
): Date[]
```

### 5.2 单元测试覆盖点（`cron.test.ts`）

| 函数 | 用例 |
|------|------|
| `parseCron` | 5 位格式解析；6 位格式解析；`*/5` 步长语法；`1-5` 范围语法；`1,3,5` 列表语法；非法格式返回 error |

- 页面可访问，标题为"Cron解析"

- 点击预设 → cron 输入框和 UI 选择器同步更新
- 修改 UI 选择器 → cron 输入框同步更新
- 输入非法格式 → 错误提示显示
- 点击清除 → 所有输入和结果清空

## 6. 开发流程

按照项目 TDD 强制流程：

1. 先写 `cron.ts` 工具函数骨架
2. 写单元测试（红阶段确认失败）
3. 实现函数（绿阶段确认通过）
4. 实现 `CronParser.vue` 组件
5. 写 E2E 测试 + axe 无障碍测试
6. 写视觉回归测试（首次运行生成 baseline）
7. 更新 `tests/config.ts` 追加 `'cron-parser-test'`
8. `npm run build` 确认编译通过
9. `npm run test && npm run test:e2e` 全部通过
