## 推荐实施顺序

### 第一批：高价值 + 简单实现

建议先做这些，快速扩充工具箱：

| 顺序 | 工具 | 分类 | 理由 |
|------|------|------|------|
| 1 | **Base64编解码** | 编码/解码 | 与URL编解码模式相同，可直接复用现有结构 |
| 2 | **UUID生成** | 随机/生成 | 极简实现，`crypto.randomUUID()` 一行搞定 |
| 3 | **Hash计算** | 安全/加密 | Web Crypto API 纯前端，实用性强 |

### 第二批：高价值 + 中等复杂度

| 顺序 | 工具 | 分类 | 理由 |
|------|------|------|------|

| 5 | **CSV ↔ JSON转换** | 数据处理 | 数据处理高频需求 |
| 6 | **Diff对比** | 开发辅助 | 需要diff算法，但实用性强 |

## 技术约束

新增工具需满足以下条件：

1. **纯前端** — 数据不上传，本地计算
2. **无后端依赖** — 不依赖第三方API（除非有fallback）
3. **工具注册** — 在 `src/tools/registry.ts` 注册，自动出现在首页
4. **目录结构** — 遵循 `src/tools/<tool-name>/` + `<ToolName>.vue` + `<tool-name>.ts` 模式
5. **UI一致** — 使用 shadcn-vue 组件（Button, Card, Select 等）
6. **Result模式** — 工具函数返回 `{ success, output, error? }` 接口
